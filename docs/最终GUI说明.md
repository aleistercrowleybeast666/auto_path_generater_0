# 最终 GUI 说明

主窗口保持三种生成模式。右侧在“固定8点 / 最优路段 / 批量”后增加第4个“Leg 模板”标签，原规划参数页顺延。

“路径编辑”包含统一场地画布、点表和动作表。画布支持 4000×2000 场地、网格、障碍物、双击、拖动、yaw 手柄、缩放和 fit。数据全部来自 V4 project/case；FULL_AUTO 图元只读。

“最优路段与批量生成”显示 Case 摘要和 leg 表，可优化/重算选中 leg、优化当前 Case 或全部缺失、清除 leg、验证当前/全部、生成当前全自动 ID 或全部 360，并显示后台进度与取消状态。MANUAL 下公共 leg 操作禁用。

“Leg 模板”显示18个与 traj_id 无关的逻辑槽位，允许编辑 XY waypoint、查看精确实例和严格验证结果，并输出三个模板 JSON。它不是新的生成模式；轻量预览不代表通过，只有点击确定后才运行后台严格验证。

工具栏依次覆盖新建/打开 V4 项目、保存项目、打开/保存当前 Case、模式、traj_id、生成、验证、导出模式 BIN、设为 final、撤销和重做。菜单提供 V3.5 导入和旧 V4 目录迁移；不存在 V3.5 导出或自动重规划按钮。
