# V3.5 迁移

通过菜单“导入旧 V3.5 工程”读取 `HJMB_PATH_EDITOR_JSON_V35`。目标只能选择 MANUAL 或 SEMI_AUTO，不能选择 FULL_AUTO。

MANUAL 迁移保留可表达的 START/WAYPOINT/ARRIVAL、yaw 和机械动作。SEMI_AUTO 仅在旧数据能明确形成 8 个逻辑锚点时转换，并将动作绑定到对应锚点。无法表达的动作逐项列为 unsupported；最终动作不满足 DROP_* STOP_AND_WAIT 时明确警告并阻止正式发布。

迁移结果始终为 V4 Case、`STALE`、未审批，需要人工复核。正式流程不会输出 V3.5 JSON/BIN，也不会导入旧 editor 或 planner。

旧 V4 输出目录使用“迁移旧 V4 输出目录”：先 dry-run，检查报告与冲突后再执行。迁移采用原子写入，不覆盖目标，也不猜测无 mode 的平铺 BIN/portable。
